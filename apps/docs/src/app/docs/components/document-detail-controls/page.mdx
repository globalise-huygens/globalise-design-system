# Document Detail Controls

Reusable controls for dense document-viewer interfaces. These are styled wrappers around React Aria primitives, so selection, checkbox, number entry, tooltip, and keyboard behavior stay consistent across the document overlay, metadata sidebar, search sidebar, and reference panels.

## Patterns

Use `DocumentDetailSegmentedToggleGroup` for compact mutually exclusive or multi-select choices, such as Scan/Text visibility, N/D transcription mode, and entity sorting. Use `selectionMode="multiple"` when more than one item can be active.

Use `DocumentDetailCheckbox` for binary filters, such as Search hits or Type grouping. It renders a React Aria checkbox rather than a button with checkbox styling.

Use `DocumentDetailNumberField` for editable numeric navigation, including scan numbers, search-hit numbers, and zoom percentages. Always provide `aria-label`, `minValue`, and `maxValue`.

Use `DocumentDetailTooltip` for all document-detail tooltips and short hover labels. Its neutral-700 surface, white/10 outline, uppercase 10px label, and layered shadow are the standard document-viewer hover style and are applied automatically. Do not hand-style tooltip surfaces in feature code; wrap the trigger in `DocumentDetailTooltip` and pass the text through `label`.

Use `DocumentDetailPopoverSurface` for richer pinned or interactive popovers, such as the content-warning message. Keep tooltips for short non-interactive hints and popovers for content that contains links, longer copy, or interactive controls.

Use `DocumentDetailReferenceCard` for repeated scan/document/object references. Keep navigation, copy, and external-link actions as separate interactive elements inside the card.
