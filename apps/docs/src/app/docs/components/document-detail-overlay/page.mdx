# Document Detail Overlay

A near-full shell overlay for document scan and transcription workflows. It composes top and bottom bars, expanded or collapsed metadata navigation, split scan/transcript panes, floating pane toolbars, and bottom pagination.

## Layout Notes

The overlay modal now uses the same shell grid contract as pages: `repeat(var(--shell-cols), minmax(0, 1fr))` with `var(--shell-gutter)`. The default desktop placement uses `gds-document-detail-dialog-placement`, a near-full 16-column shell span, so the viewer occupies most of the viewport while still leaving edge context.

Use `contentClassName` to replace shell placement for alternate page or dual-view variants. Use `dialogClassName` for visual styling on the frame itself.

The component family is intentionally split into reusable primitives: bars (`DocumentDetailTopBar`, `DocumentDetailBottomBar`, `DocumentDetailBarGroup`, `DocumentDetailTitle`), controls (`DocumentDetailControl`, `DocumentDetailToolButton`, `DocumentDetailRailButton`, `DocumentDetailSegmentedControl`, `DocumentDetailSegment`), navigation (`DocumentDetailMetadataSidebar`, `DocumentDetailMetadataSidebarButton`, `DocumentDetailMetadataSidebarBadge`, `DocumentDetailIconRail`), frame regions (`DocumentDetailBody`, `DocumentDetailViewer`, `DocumentDetailSplitViewer`, `DocumentDetailViewerPane`, `DocumentDetailSidePanel`), and content helpers (`DocumentDetailToolbar`, `DocumentDetailFloatingToolbar`, `DocumentDetailCanvas`, `DocumentDetailTranscriptCanvas`, `DocumentDetailTranscriptLine`, `DocumentDetailPanelHeader`).
