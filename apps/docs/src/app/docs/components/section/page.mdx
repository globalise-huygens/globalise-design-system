# Section & SectionDivider

Layout primitives for dividing pages into background‑coloured bands with consistent vertical spacing, and inserting a styled divider between them.

### Section with Grid mode

Use this mode for explicit column placement:

<CodeBlock
  code={`<Section background="dark" spacing="large">
  <Grid className="mx-auto w-full max-w-shell-max px-shell-margin">
    <div className="slot-content-band">

    </div>
    <aside className="slot-content-right">Secondary rail</aside>
  </Grid>
</Section>`}
/>

## Composition rules

Use one of these two patterns per section:

| Pattern                       | Use for               | Default |
| ----------------------------- | --------------------- | ------- |
| `Section` + `withContainer`   | Linear content blocks | Yes     |
| `Section` + `Grid` + `slot-*` | Multi-region layouts  | Yes     |

Avoid mixing both wrappers in one section unless you intentionally need an inner bounded region.

## Spacing policy

`Section` owns vertical rhythm through `spacing` (`default` and `large`). Inside each section, use strict spacing defaults:

| Context                                 | Default token/class                        |
| --------------------------------------- | ------------------------------------------ |
| Between major blocks in a section       | `gap-s24`                                  |
| Between tightly related rows            | `gap-s12`                                  |
| Internal panel/card padding             | `p-s24`                                    |
| Extra separation between section groups | `mt-s24` on the next section content block |

Use `SectionDivider` for visual transitions when separation should be semantic, not only spatial.
