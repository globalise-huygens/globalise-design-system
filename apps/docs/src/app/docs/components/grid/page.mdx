# Grid

The page-level layout grid following the Figma grid specification: **12 columns, stretch, auto width, 0 margin, 0 gutter**. At a max-width of 1440px each column is exactly 120px wide.

Most content is placed in **columns 2–11** (use `col-start-2 col-span-10` or `Container` for this). Full-bleed elements span all 12 columns. Items placed in column 1 or 12 sit at the very edge of the page.

## Preview

<ComponentPreview>
  <Grid className="w-full max-w-360">
    {Array.from({ length: 12 }).map((_, i) => (
      <div
        key={i}
        className={`h-20 flex items-center justify-center text-xs font-mono ${
          i === 0 || i === 11
            ? "bg-white/5 text-white/30"
            : "bg-brand-turquoise/20 text-white/50"
        }`}
      >
        {i + 1}
      </div>
    ))}
  </Grid>
</ComponentPreview>

Columns 1 and 12 (dimmed) are the edge columns. Columns 2–11 (highlighted) are the standard content area, equivalent to using `Container`.

## Usage

<CodeBlock code={`import { Grid } from "@globalise/design-system";

{/* Page-level grid with max-width */}

<Grid className="mx-auto max-w-360">
  {/* Standard content area (columns 2–11) */}
  <div className="col-span-12 px-4 sm:col-start-2 sm:col-span-10 sm:px-0">
    <p>Content sits in the standard 10-column content area.</p>
  </div>
</Grid>

{/* Full bleed */}

<Grid className="mx-auto max-w-360">
  <div className="col-span-12">Full-width element</div>
</Grid>

{/* Two-column layout within the content area */}

<Grid className="mx-auto max-w-360">
  <div className="col-span-12 px-4 sm:col-start-2 sm:col-span-5 sm:px-0">
    Left column
  </div>
  <div className="col-span-12 px-4 sm:col-start-7 sm:col-span-5 sm:px-0">
    Right column
  </div>
</Grid>`} />

## Grid specification

| Property               | Value   |
| ---------------------- | ------- |
| Columns                | 12      |
| Type                   | Stretch |
| Width                  | Auto    |
| Margin                 | 0       |
| Gutter                 | 0       |
| Max width              | 1440px  |
| Column width at 1440px | 120px   |

## Responsive breakpoints

The grid is always 12 columns at every viewport width. Responsive behaviour is achieved by changing `col-span-*` and `col-start-*` on child elements at each breakpoint.

| Breakpoint | Width    | Typical layout               | Mobile padding   |
| ---------- | -------- | ---------------------------- | ---------------- |
| Default    | < 640px  | Single column, `col-span-12` | `px-4` (16px)    |
| `sm`       | ≥ 640px  | Content area cols 2–11       | none (grid cols) |
| `md`       | ≥ 768px  | Content area cols 2–11       | none             |
| `lg`       | ≥ 1024px | Multi-column layouts         | none             |
| `xl`       | ≥ 1280px | Full 12-column grid          | none             |

### Mobile-first column pattern

On mobile, children span all 12 columns with padding. At `sm` and up, they snap into named grid columns:

<CodeBlock
  code={`{/* Standard content area — mobile-safe */}
<Grid className="mx-auto max-w-360">
  <div className="col-span-12 px-4 sm:col-start-2 sm:col-span-10 sm:px-0">
    Content here
  </div>
</Grid>`}
/>

### Responsive multi-column example

<CodeBlock
  code={`<Grid className="mx-auto max-w-360">

## Component placement map

How each component in the design system should sit within the 12-column grid:

### Full-bleed components (cols 1–12)

| Component        | Grid placement   | Notes                                                              |
| ---------------- | ---------------- | ------------------------------------------------------------------ |
| `Navbar`         | `col-span-12`    | Full width, internally constrains to `max-w-[1440px]`              |
| `Section`        | Not a grid child | Wraps the `Grid` — provides background colour and vertical spacing |
| `CardHero` strip | `col-span-12`    | Horizontally scrollable, edge-to-edge                              |

### Standard content area (cols 2–11)

| Component           | Desktop grid placement                  | Tablet (sm)               | Mobile             |
| ------------------- | --------------------------------------- | ------------------------- | ------------------ |
| `Container`         | Equivalent to `col-start-2 col-span-10` | Same                      | `col-span-12 px-4` |
| `CardFeatured`      | `col-start-2 col-span-10`               | Same                      | `col-span-12 px-4` |
| `ArticleRow`        | `col-start-2 col-span-10`               | Same, cards stack         | `col-span-12 px-4` |
| `NewsletterSignup`  | `col-start-2 col-span-10`               | Same                      | `col-span-12 px-4` |
| `Divider`           | `col-start-2 col-span-10`               | Same                      | `col-span-12 px-4` |
| `Typography` (hero) | `col-start-2 col-span-7`                | `col-start-2 col-span-10` | `col-span-12 px-4` |

### Multi-column content layouts

| Layout           | Left column              | Right column             | Tablet                           | Mobile                    |
| ---------------- | ------------------------ | ------------------------ | -------------------------------- | ------------------------- |
| Text + pullquote | `col-start-2 col-span-5` | `col-start-8 col-span-4` | Both → `col-start-2 col-span-10` | Both → `col-span-12 px-4` |
| Two equal halves | `col-start-2 col-span-5` | `col-start-7 col-span-5` | Same                             | Both → `col-span-12 px-4` |

### Card grids (nested within content area)

Cards that form a grid of items use a nested sub-grid within the content area:

<CodeBlock
  code={`<Grid className="mx-auto max-w-360">
  <div className="col-span-12 px-4 sm:col-start-2 sm:col-span-10 sm:px-0">
    {/* Sub-grid: 1 col → 2 cols → 4 cols */}
    <div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-0">
      <CardGlance ... />
      <CardGlance ... />
      <CardGlance ... />
      <CardGlance ... />
    </div>
  </div>
</Grid>`}
/>

| Card component     | Desktop                          | Tablet (sm)       | Mobile            |
| ------------------ | -------------------------------- | ----------------- | ----------------- |
| `CardGlance` (×4)  | 4 across in content area         | 2 × 2             | Stacked           |
| `CardArticle` (×4) | 4 across via `ArticleRow`        | 4 across          | Stacked           |
| `CardHero` strip   | Horizontal scroll, full bleed    | Horizontal scroll | Horizontal scroll |
| `CardFeatured`     | Accordion/expand in content area | Same              | Same, stacked     |

## Grid vs Container

`Grid` and `Container` serve complementary roles:

|              | `Grid`                                                  | `Container`                                        |
| ------------ | ------------------------------------------------------- | -------------------------------------------------- |
| **Purpose**  | 12-column placement system                              | Quick content-area wrapper                         |
| **Output**   | `grid grid-cols-12`                                     | `max-w-[1440px]` + padding                         |
| **Columns**  | Explicit via `col-start-*` / `col-span-*`               | Implicit (cols 2–11 via padding)                   |
| **Use when** | You need multi-column layout or explicit column control | Single-column content in the standard content area |
| **Nesting**  | Can nest sub-grids for card layouts                     | Can contain `Grid` for sub-layouts                 |

For the example site, `Grid` is used as the primary layout mechanism in every section, replacing `Container` at the page level. `Container` remains useful for simpler layouts where explicit column control isn't needed.
