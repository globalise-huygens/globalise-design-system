# Grid

The page-level layout grid now follows the shared shell contract: **4 columns (mobile), 8 columns (tablet), 16 columns (desktop), stretch, 1440px max width, 16px gutters**.

Use `col-span-full` as the safe default at every breakpoint. Use explicit `col-start-*` and `col-span-*` at `xl` when you need placement inside the 16-column desktop shell.

## Preview

<ComponentPreview>
  <ResponsiveGridPreview />
</ComponentPreview>

Dimmed columns are **outer shell columns**, not disabled space. On desktop this is 1-2 and 15-16; on tablet this becomes 1 and 8; on mobile all columns are in-band.

Overlays and modals can still occupy the full shell when needed (`slot-full-bleed`) or stay in the content band (`slot-content-band`).

## Grid specification

| Property              | Value        |
| --------------------- | ------------ |
| Columns               | 4 / 8 / 16   |
| Type                  | Stretch      |
| Max width             | 1440px       |
| Desktop page margin   | 32px         |
| Mobile page margin    | 16px         |
| Gutter                | 16px         |
| Standard content band | Columns 3-14 |

## Responsive breakpoints

The grid is responsive by token: 4 columns at mobile, 8 at tablet, and 16 at desktop. Responsive behavior comes from using `col-span-full` by default and applying `xl:col-start-*` / `xl:col-span-*` for 16-column placement.

| Breakpoint  | Typical layout                                        |
| ----------- | ----------------------------------------------------- |
| Default     | Full-width children using `col-span-full`             |
| `xl` and up | Centered content band using `col-start-3 col-span-12` |

### Mobile-first column pattern

On smaller screens, let children span all 16 columns. At desktop, snap them into the centered content band:

## Component placement map

How the shared components map onto the 16-column shell:

### Full-bleed components (cols 1-16)

| Component        | Grid placement   | Notes                                     |
| ---------------- | ---------------- | ----------------------------------------- |
| `Navbar`         | `col-span-full`  | Full-width shell aligned to page margins  |
| `Section`        | Not a grid child | Wraps the `Grid` and owns vertical rhythm |
| `CardHero` strip | `col-span-full`  | Edge-to-edge within the page shell        |

### Standard content band (cols 3-14)

| Component           | Desktop grid placement                   | Mobile            |
| ------------------- | ---------------------------------------- | ----------------- |
| `Container`         | Equivalent to the centered 12-track band | Page margins only |
| `CardFeatured`      | `xl:col-start-3 xl:col-span-12`          | `col-span-full`   |
| `ArticleRow`        | `xl:col-start-3 xl:col-span-12`          | `col-span-full`   |
| `NewsletterSignup`  | `xl:col-start-3 xl:col-span-12`          | `col-span-full`   |
| `Divider`           | `xl:col-start-3 xl:col-span-12`          | `col-span-full`   |
| `Typography` (hero) | `xl:col-start-3 xl:col-span-8`           | `col-span-full`   |

### Multi-column content layouts

| Layout           | Left column                    | Right column                    | Mobile               |
| ---------------- | ------------------------------ | ------------------------------- | -------------------- |

## Grid vs Container

`Grid` and `Container` serve complementary roles:

|              | `Grid`                                          | `Container`                                    |
| ------------ | ----------------------------------------------- | ---------------------------------------------- |
| **Purpose**  | 16-column placement system                      | Quick wrapper for the centered content band    |
| **Output**   | Shell grid with shared gutters                  | Max width plus shared page margins             |
| **Columns**  | Explicit via `col-start-*` / `col-span-*`       | Implicit via padding and width                 |
| **Use when** | You need explicit shell or multi-column control | You need a fast, single-column content wrapper |
| **Nesting**  | Can nest sub-grids for card layouts             | Can contain `Grid` for sub-layouts             |

For the example site, `Grid` is used as the primary layout mechanism in every section, replacing `Container` at the page level. `Container` remains useful for simpler layouts where explicit column control isn't needed.

## Grid + Section + Container contract

Use this responsibility split across all page layouts:

| Primitive   | Owns                                                  | Does not own           |
| ----------- | ----------------------------------------------------- | ---------------------- |
| `Section`   | Vertical rhythm (`spacing`) + background band         | Column placement       |
| `Grid`      | Column placement and slot composition                 | Section spacing policy |
| `Container` | Centered width + horizontal insets for simple content | Multi-column placement |

### Canonical pattern

Use `Section` as the outer band, then place content with `Grid` slots:

<CodeBlock
  code={`<Section background="dark" spacing="large">
  <Grid className="mx-auto w-full max-w-shell-max px-shell-margin">
    <div className="slot-content-band">Main content</div>
    <aside className="slot-content-right">Supporting content</aside>
  </Grid>
</Section>`}
/>

### Do / do not

| Do                                                                                                       | Do not                                                                                     |
| -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| Use `slot-*` classes for repeatable placement (`slot-content-band`, `slot-left-rail`, `slot-right-rail`) | Mix ad-hoc `xl:col-start-*` / `xl:col-span-*` values across pages for the same layout role |
| Keep `Section` outside `Grid`                                                                            | Put `Section` as a grid child                                                              |
| Use `Container` for simple single-column sections                                                        | Nest `Container` inside `Grid` slots unless you intentionally need an inner width cap      |

### Spacing defaults when Grid is used

- Section-to-section rhythm: use adjacent `Section` blocks first; if extra space is needed, use `mt-s24` on the next section's first grid child.
- Major in-band spacing inside a section: `gap-s24`.
- Tight component row spacing: `gap-s12`.
- Panel/card internal padding: `p-s24`.
