# Grid

The page-level layout grid now follows the shared shell contract: **16 columns, stretch, 32px desktop margins, 16px gutters**. At a max width of 1440px, the standard content band is the centered **12-track span from columns 3â€“14**.

Use the full 16-column grid for shell composition and alignment. Use the centered 12-track band for most content, cards, and overlays.

## Grid specification

| Property              | Value        |
| --------------------- | ------------ |
| Columns               | 16           |
| Type                  | Stretch      |
| Max width             | 1440px       |
| Desktop page margin   | 32px         |
| Mobile page margin    | 16px         |
| Gutter                | 16px         |
| Standard content band | Columns 3-14 |

## Responsive breakpoints

The grid stays 16 columns across breakpoints. Responsive behavior comes from changing `col-span-*` and `col-start-*` values while the page margins and standard content band stay consistent.

| Breakpoint  | Typical layout                                        |
| ----------- | ----------------------------------------------------- |
| Default     | Full-width children using `col-span-16`               |
| `lg` and up | Centered content band using `col-start-3 col-span-12` |

### Mobile-first column pattern

## Component placement map

How the shared components map onto the 16-column shell:

### Full-bleed components (cols 1-16)

| Component        | Grid placement   | Notes                                     |
| ---------------- | ---------------- | ----------------------------------------- |
| `Navbar`         | `col-span-16`    | Full-width shell aligned to page margins  |
| `Section`        | Not a grid child | Wraps the `Grid` and owns vertical rhythm |
| `CardHero` strip | `col-span-16`    | Edge-to-edge within the page shell        |

### Standard content band (cols 3-14)

| Component           | Desktop grid placement                   | Mobile            |
| ------------------- | ---------------------------------------- | ----------------- |
| `Container`         | Equivalent to the centered 12-track band | Page margins only |
| `CardFeatured`      | `col-start-3 col-span-12`                | `col-span-16`     |
| `ArticleRow`        | `col-start-3 col-span-12`                | `col-span-16`     |
| `NewsletterSignup`  | `col-start-3 col-span-12`                | `col-span-16`     |
| `Divider`           | `col-start-3 col-span-12`                | `col-span-16`     |
| `Typography` (hero) | `col-start-3 col-span-8`                 | `col-span-16`     |

### Multi-column content layouts

| Layout           | Left column              | Right column              | Mobile             |
| ---------------- | ------------------------ | ------------------------- | ------------------ |
| Text + pullquote | `col-start-3 col-span-7` | `col-start-11 col-span-4` | Both `col-span-16` |
| Two equal halves | `col-start-3 col-span-6` | `col-start-9 col-span-6`  | Both `col-span-16` |

## Grid vs Container

`Grid` and `Container` serve complementary roles:

|              | `Grid`                                          | `Container`                                    |
| ------------ | ----------------------------------------------- | ---------------------------------------------- |
| **Purpose**  | 16-column placement system                      | Quick wrapper for the centered content band    |
| **Output**   | Shell grid with shared gutters                  | Max width plus shared page margins             |
| **Columns**  | Explicit via `col-start-*` / `col-span-*`       | Implicit via padding and width                 |
| **Use when** | You need explicit shell or multi-column control | You need a fast, single-column content wrapper |
| **Nesting**  | Can nest sub-grids for card layouts             | Can contain `Grid` for sub-layouts             |

For the example site, `Grid` is used as the primary layout mechanism in every section, replacing `Container` at the page level. `Container` remains useful for simpler layouts where explicit column control isn't needed.
