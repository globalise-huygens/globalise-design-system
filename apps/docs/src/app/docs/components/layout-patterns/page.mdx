# Layout Patterns

Canonical recipes for composing `Section`, `Grid`, and `Container` with consistent spacing defaults.

## Primitive contract

Use this contract on all pages:

| Primitive   | Responsibility                                             |
| ----------- | ---------------------------------------------------------- |
| `Section`   | Background band and vertical rhythm                        |
| `Grid`      | Column placement in the shell                              |
| `Container` | Simple centered wrapper when no column placement is needed |

## Pattern 1: Section + Grid (main + rail)

Use for complex page regions.

<CodeBlock
  code={`<Section background="dark" spacing="large">
  <Grid className="mx-auto w-full max-w-shell-max px-shell-margin">
    <div className="slot-content-band flex flex-col gap-s24">
      <Typography variant="h2" className="text-brand-white">Heading</Typography>
      <Typography variant="p" className="text-brand-white/70">Lead text</Typography>
    </div>
    <aside className="slot-content-right flex flex-col gap-s12">
      <CardBase className="p-s24">Related content</CardBase>
    </aside>
  </Grid>
</Section>`}
/>

Defaults in this pattern:

- Major content rhythm: `gap-s24`
- Tight related rows: `gap-s12`
- Card/panel internal padding: `p-s24`

## Pattern 2: Section withContainer (simple content)

Use for linear content blocks (copy, forms, CTA groups) where explicit columns are unnecessary.

<CodeBlock
  code={`<Section background="light" spacing="default" withContainer>
  <div className="flex flex-col gap-s24">
    <Typography variant="h3">Newsletter</Typography>
    <NewsletterSignup />
  </div>
</Section>`}
/>

## Pattern 3: Adjacent section transitions

Use `Section` spacing first. Add extra top margin only when a transition needs additional separation.

<CodeBlock
  code={`<Section background="dark" spacing="default" withContainer>
  <Typography variant="h2" className="text-brand-white">Overview</Typography>
</Section>

<Section background="light" spacing="default">
  <Grid className="mx-auto w-full max-w-shell-max px-shell-margin">
    <div className="slot-content-band mt-s24">
      <Typography variant="h3">Details</Typography>
    </div>
  </Grid>
</Section>`}
/>

## Anti-patterns

| Anti-pattern                                  | Why it hurts                                                   | Preferred approach                                     |
| --------------------------------------------- | -------------------------------------------------------------- | ------------------------------------------------------ |
| `Section` -> `Grid` -> `Container`            | Double wraps horizontal constraints and causes alignment drift | Use either `Grid` slots or `withContainer`             |
| Ad-hoc `col-start-*` values repeated per page | Layout roles become inconsistent and hard to maintain          | Use shared `slot-*` classes                            |
| Random `mt-*` spacing between siblings        | Vertical rhythm drifts between pages                           | Use `spacing` on `Section`, then `gap-s24` / `gap-s12` |
