# Container

Applies shared horizontal structure (max width + insets) without requiring explicit grid-column placement. Use it when you want predictable centered content quickly.

You still need `Container` for simple wrappers and single-column sections. Use `Grid` when you need explicit multi-column slot placement.

## Responsive padding

Container uses the shared page-margin tokens:

| Breakpoint  | Padding |
| ----------- | ------- |
| Default     | `16px`  |
| `lg` and up | `32px`  |

At the page max width, that aligns Container with the same centered content band used by the 16-column grid.

## Container vs Grid decision

Use this table as the default rule:

| Use `Container` when...                                  | Use `Grid` when...                                                       |
| -------------------------------------------------------- | ------------------------------------------------------------------------ |
| You need a centered single-column wrapper                | You need explicit column placement                                       |
| Section content is mostly linear (text, CTA, form stack) | You have multiple regions (main + side, asymmetric cards, rails)         |
| You want width/inset consistency without slot logic      | You need named slot composition (`slot-content-band`, `slot-right-rail`) |

## Interaction rules with Section and Grid

| Pattern                  | Recommendation                                      |
| ------------------------ | --------------------------------------------------- |
| `Section` -> `Container` | Preferred for simple blocks                         |
| `Section` -> `Grid`      | Preferred for multi-region layouts                  |
| `Section withContainer`  | Preferred when Section should own wrapping directly |
| `Grid` -> `Container`    | Avoid by default (double wrapping)                  |

### Do / do not

- Do use one outer horizontal wrapper per section (`Container` or `Grid`, not both).
- Do keep spacing between direct children on tokens (`gap-s24` for major blocks, `gap-s12` for tight groups).
- Do not add extra horizontal padding classes on `Container` children to compensate for layout; change `size` or `inset` instead.
